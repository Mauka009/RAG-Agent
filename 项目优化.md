1.使用 .env 文件存储密钥，不把 API 密钥硬编码在代码里。

2.安全遍历history，如果history中混入额外字段也不会解包报错。通过 isinstance 做了运行时类型检查，分别适配了字典模式（新版 Gradio）和列表/元组模式（旧版 Gradio）。这种设计模式叫适配器模式（Adapter Pattern），它使得业务逻辑（调用大模型）与前端框架的版本变化彻底解耦，即便以后 Gradio 升级，核心代码也无需改动。

3.建config.py用来把程序中所有"会变动的值"单独放到一个地方管理，如果它们散落在代码各个角落，每次改都要到处找，很麻烦。

4.异步调用：通过减少等待（比如在等待 API 返回的 2 秒里，可以同时处理另一个用户的请求，或执行其他计算），减少总耗时（考虑到本项目仅测试单用户问答，影响不大。无更改）。

5.优化读取，从只能读txt文件到自动识别文件类型，支持：TXT、PDF（文字型）、JPG/PNG（图片OCR）。

6.实现单文档和多文档同时处理二者间的切换，测试用单文档增加效率。

7.简历缓存重建机制，通过配置开关控制是否强制刷新向量库，避免出现换了新的文档但加载本地向量库。

8.引入 RAGAS 自动化评估框架为优化提供数据支撑和方向，方便迭代。
